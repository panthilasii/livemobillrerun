# vcam-app — Android receiver (Phase 4a)

> **Built and installable.** Gradle project + manifest + Kotlin sources are
> in. `tools/bootstrap_build_macos.sh` sets up a portable JDK 17 + Android
> SDK + Gradle 8.10.2 toolchain in `../.tools/` and lets you build the APK
> without Android Studio.

## Quick build (no Android Studio)

```bash
cd vcam-app
bash tools/bootstrap_build_macos.sh   # one-time, ~600 MB into ../.tools/
source ../.tools/env.sh               # JAVA_HOME, ANDROID_HOME, PATH, LANG=C
gradle :app:assembleDebug             # ~1 min on M-series
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

The `LANG=C` is **mandatory** on Thai-locale macOS — otherwise Java picks
`BuddhistCalendar`, sees year 2569, and AGP's zip writer fails the
`Verify.verify(year ≤ 2107)` check inside `MsDosDateTimeUtils.packDate`.
`env.sh` exports it for you.

## Quick build (with Android Studio)

Open `vcam-app/` as a project. Studio will prompt to download Gradle
8.10.2 + AGP 8.7.3. Sync, then **Build → Build Bundle(s) / APK(s) → Build
APK(s)**.

## Purpose

Connects to the PC streamer over `localhost:8888` (via `adb reverse`),
decodes the H.264 stream, writes the decoded YUV frames to
`/data/local/tmp/vcam.yuv` for the Magisk HAL hook (or LSPosed module) to
serve to other apps as the camera feed.

## Stack

| Choice              | Why                                                |
| ------------------- | -------------------------------------------------- |
| **Kotlin**          | Standard Android language                          |
| **MediaCodec async**| Lowest-latency H.264 decoder available on Android  |
| **Foreground svc**  | Survive in background, app-killer resistant        |
| **Plain Sockets**   | One TCP connection, no need for OkHttp/Retrofit    |
| **No DI framework** | Project too small; manual wiring                   |

## Pipeline contract

### Input (TCP)

- Host: `127.0.0.1`
- Port: `8888`
- Format: Annex-B H.264 (start codes `00 00 00 01` between NAL units)
- Reconnect: 1 s backoff if disconnected
- Buffer: at least 256 KB ring buffer between socket and MediaCodec

### Output (file)

- Path: `/data/local/tmp/vcam.yuv`
- Size: `16 + width * height * 3 / 2` bytes (header + YUV420 I420)
- Header (16 bytes, little-endian):
  ```
  uint32  magic  = 0x564D4143  ('VCAM')
  uint32  width
  uint32  height
  uint32  frame_counter (monotonically increasing)
  ```
- Update: write to `vcam.yuv.tmp` then `rename(2)` for atomic swap
- Cadence: same as decoder output (~30 fps)

## Reference implementation snippets

### TcpClient.kt

```kotlin
class TcpClient(
    private val host: String,
    private val port: Int,
    private val onBytes: (ByteArray, Int) -> Unit,
) {
    @Volatile private var running = false
    private var thread: Thread? = null

    fun start() {
        running = true
        thread = Thread { loop() }.apply { isDaemon = true; start() }
    }

    fun stop() {
        running = false
        thread?.interrupt()
    }

    private fun loop() {
        val buf = ByteArray(64 * 1024)
        while (running) {
            try {
                Socket().use { sock ->
                    sock.connect(InetSocketAddress(host, port), 3000)
                    val input = sock.getInputStream()
                    while (running) {
                        val n = input.read(buf)
                        if (n <= 0) break
                        onBytes(buf, n)
                    }
                }
            } catch (e: Exception) {
                Log.w(TAG, "tcp dropped: ${e.message}")
                Thread.sleep(1000)
            }
        }
    }
    companion object { const val TAG = "TcpClient" }
}
```

### H264Decoder.kt (async MediaCodec)

```kotlin
class H264Decoder(
    private val width: Int,
    private val height: Int,
    private val onYuvFrame: (ByteArray, Int, Int) -> Unit,
) {
    private lateinit var codec: MediaCodec

    fun start() {
        val mime = MediaFormat.MIMETYPE_VIDEO_AVC
        val format = MediaFormat.createVideoFormat(mime, width, height).apply {
            setInteger(
                MediaFormat.KEY_COLOR_FORMAT,
                MediaCodecInfo.CodecCapabilities.COLOR_FormatYUV420Flexible
            )
        }
        codec = MediaCodec.createDecoderByType(mime).apply {
            setCallback(callback)
            configure(format, /*surface=*/ null, /*crypto=*/ null, 0)
            start()
        }
    }

    fun feed(data: ByteArray, len: Int) {
        // queued asynchronously via callback.onInputBufferAvailable
        pendingInput.offer(data.copyOf(len))
    }

    fun stop() {
        codec.stop(); codec.release()
    }

    private val pendingInput = LinkedBlockingQueue<ByteArray>()
    private val callback = object : MediaCodec.Callback() {
        override fun onInputBufferAvailable(c: MediaCodec, idx: Int) {
            val payload = pendingInput.poll(50, TimeUnit.MILLISECONDS) ?: return
            val buf = c.getInputBuffer(idx)!!
            buf.clear(); buf.put(payload)
            c.queueInputBuffer(idx, 0, payload.size, System.nanoTime() / 1000, 0)
        }

        override fun onOutputBufferAvailable(
            c: MediaCodec, idx: Int, info: MediaCodec.BufferInfo
        ) {
            val image = c.getOutputImage(idx) ?: run {
                c.releaseOutputBuffer(idx, false); return
            }
            val yuv = imageToI420(image)
            onYuvFrame(yuv, image.width, image.height)
            c.releaseOutputBuffer(idx, false)
        }
        override fun onError(c: MediaCodec, e: MediaCodec.CodecException) {
            Log.e("H264Decoder", "codec error", e)
        }
        override fun onOutputFormatChanged(c: MediaCodec, f: MediaFormat) { }
    }
}
```

### YuvFileWriter.kt

```kotlin
object YuvFileWriter {
    private const val MAGIC = 0x564D4143  // 'VCAM'
    private const val PATH = "/data/local/tmp/vcam.yuv"
    private val frameCounter = AtomicInteger(0)

    fun write(yuv: ByteArray, width: Int, height: Int) {
        val tmp = File("$PATH.tmp")
        val fc = frameCounter.incrementAndGet()
        DataOutputStream(FileOutputStream(tmp)).use { out ->
            // header — little-endian 16 bytes
            out.writeIntLE(MAGIC)
            out.writeIntLE(width)
            out.writeIntLE(height)
            out.writeIntLE(fc)
            out.write(yuv)
        }
        // atomic swap
        tmp.renameTo(File(PATH))
    }

    private fun DataOutputStream.writeIntLE(v: Int) {
        write(v and 0xff); write((v ushr 8) and 0xff)
        write((v ushr 16) and 0xff); write((v ushr 24) and 0xff)
    }
}
```

## How to install (after building)

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
adb shell am start-foreground-service \
    -n com.livemobillrerun.vcam/.VcamService
adb reverse tcp:8888 tcp:8888
```

Open the app once to grant the notification permission, then it can run
in the background.

## Acceptance test

Run the app, look at logcat:

```bash
adb logcat | grep -E 'TcpClient|H264Decoder|YuvFileWriter'
```

Then on the phone:

```bash
adb shell ls -la /data/local/tmp/vcam.yuv
adb shell stat -c '%y %s' /data/local/tmp/vcam.yuv
```

The mtime should update ~30 times per second and size should equal
`16 + 1280 * 720 * 3 / 2` = `16 + 1382400` = `1382416` bytes (default
1280×720 / I420).
