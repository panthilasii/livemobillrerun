## 0. เครื่องที่รองรับ

> **อ่านก่อนซื้อ** — ระบบ NP Create ใช้กับ **มือถือ Android**
> เท่านั้น **ไม่รองรับ iPhone / iPad / iOS** ทุกรุ่น

### สเปคขั้นต่ำ

| รายการ | ขั้นต่ำ | แนะนำ |
|---|---|---|
| **Android** | 8.0 (Oreo) | 11 ขึ้นไป |
| **CPU** | ARM64 (arm64-v8a) | ARM64 |
| **RAM** | 3 GB | 4 GB ขึ้นไป |
| **พื้นที่ว่าง** | 2 GB | 4 GB ขึ้นไป |
| **USB Debugging** | ต้องเปิดได้ | — |

> เครื่องที่ผลิตหลังปี 2018 เกือบทั้งหมดผ่านสเปคขั้นต่ำ

### ✅ แบรนด์ที่รองรับ (เทสจริง / ใช้งานได้ดี)

| แบรนด์ | OS / ROM | หมายเหตุ |
|---|---|---|
| **Xiaomi / Redmi / POCO** | MIUI / HyperOS | ✅ ทดสอบจริงกับ Redmi 14C |
| **Samsung Galaxy** (A/M/F/S/Note/Z) | One UI 4 ขึ้นไป | ปิด Knox / Secure Folder ก่อนติดตั้ง |
| **OPPO** (A/Reno/Find) | ColorOS | ใช้ได้ปกติ |
| **Realme** (Number/GT/Narzo) | RealmeOS / ColorOS | ใช้ได้ปกติ |
| **OnePlus** (Nord/Number) | OxygenOS | ใช้ได้ปกติ |
| **Vivo / iQOO** | OriginOS / FuntouchOS | ปิด Power Saver โหมดลึก |
| **Google Pixel** | Stock Android | ใช้ได้ดีที่สุด (ใกล้ AOSP) |
| **Asus / ROG Phone** | ZenUI | ใช้ได้ปกติ |
| **Sony Xperia** | Stock-ish | ใช้ได้ปกติ |
| **Nokia / HMD** | Android One | ใช้ได้ปกติ |
| **Motorola** | My UX (near-stock) | ใช้ได้ปกติ |

### ⚠️ ใช้ได้แต่ต้องเช็คเพิ่ม

| ประเภท | สิ่งที่ต้องดู |
|---|---|
| **Huawei รุ่นเก่า** (P30 / Mate 20–30 / Y9 ฯลฯ ใช้ EMUI) | ส่วนใหญ่ใช้ได้ — แต่ Google Services อาจไม่ครบทำให้ TikTok บางฟีเจอร์ไม่ทำงาน |
| **Honor** (รุ่นแยกจาก Huawei ใช้ MagicOS) | คล้าย Huawei เก่า ตรวจ Google Services ก่อน |
| **เครื่องที่ Root อยู่แล้ว** | ปิด Magisk Hide ของ TikTok ก่อน Patch |
| **เครื่องประกอบ / Custom ROM** (LineageOS / Pixel Experience ฯลฯ) | ต้องมี Java VM ครบ และไม่ strip Xposed compatibility |

### ❌ ไม่รองรับ

| ประเภท | สาเหตุ |
|---|---|
| **iPhone / iPad / iOS ทุกรุ่น** | iOS ไม่อนุญาตให้ติดตั้ง app ที่แก้ไขโดยไม่ Jailbreak |
| **Huawei HarmonyOS NEXT** (Mate 60 / Pura 70 / รุ่นจีน 2024+ ที่ตัด Android ออก) | ไม่ใช่ Android อีกต่อไป — kernel HarmonyOS ของตัวเอง |
| **Android 7.0 หรือต่ำกว่า** | TikTok เวอร์ชันใหม่ไม่ติดตั้งได้ |
| **เครื่อง 32-bit** (CPU armeabi-v7a only — รุ่นเก่ามาก) | TikTok 35.x+ ส่ง APK เฉพาะ ARM64 |
| **Android Tablet หลายรุ่น** | TikTok Live บน tablet จำกัดบางฟีเจอร์ตั้งแต่ฝั่ง TikTok เอง |
| **Smart TV / Android TV / Watch** | ไม่ใช่กลุ่มที่รองรับ TikTok Live |

### วิธีตรวจมือถือก่อนซื้อ — 30 วินาที

1. ไปที่ **ตั้งค่า → เกี่ยวกับโทรศัพท์**
2. เช็ค 3 จุด:
   - **Android version**: ต้อง ≥ 8.0
   - **RAM**: ต้อง ≥ 3 GB
   - **พื้นที่ว่าง**: เหลือ ≥ 2 GB
3. ส่งภาพหน้าจอมาให้แอดมิน (Line @npcreate) เช็คให้ฟรี

> **ถ้าไม่แน่ใจ ส่งรูปหน้าตั้งค่ามาให้แอดมินดูก่อนได้ครับ
> เช็คฟรี ภายใน 5 นาที**

## 1. เตรียมก่อนติดตั้ง

### บนคอมพิวเตอร์ (Windows หรือ macOS)

| สิ่งที่ต้องมี | รายละเอียด |
|---|---|
| **Python 3.13** | ดาวน์โหลดที่ [python.org/downloads](https://www.python.org/downloads) <br> **Windows สำคัญมาก:** ตอนติดตั้ง ต้องติ๊ก ☑ "Add Python 3.13 to PATH" ก่อนกด Next |
| **Internet** | ใช้แค่ตอนติดตั้งครั้งแรก (~30 วินาที) เพื่อโหลดส่วนประกอบ |
| **พื้นที่ว่างในฮาร์ดดิสก์** | อย่างน้อย 1 GB |
| **พอร์ต USB** | สำหรับเสียบมือถือครั้งแรก (หลังจากนั้นใช้ WiFi ได้) |

### บนมือถือ Android (รองรับ Android 9+)

ทำครั้งเดียวต่อเครื่อง:

1. **เปิด Developer Options**
   - ไปที่ ตั้งค่า → เกี่ยวกับโทรศัพท์ → กดที่ "Build number" / "MIUI version" 7 ครั้ง
   - มือถือจะแจ้ง "คุณเป็นนักพัฒนาแล้ว"

2. **เปิด USB Debugging**
   - ไปที่ ตั้งค่า → ระบบ → ตัวเลือกนักพัฒนา (Developer Options)
   - เปิดสวิตช์ "USB debugging" / "การดีบัก USB"

3. **ปิดแอป TikTok ตัวเดิม** (ถ้ามี)
   - กดค้าง icon TikTok → ลบออก
   - ระบบจะติดตั้ง TikTok เวอร์ชันที่แก้ไขให้ใหม่

> **หมายเหตุ Redmi/Xiaomi/POCO:** ต้องเปิดเพิ่มอีก 1 ตัวเลือก คือ
> **"Install via USB"** และ **"USB debugging (Security settings)"**
> ในเมนู Developer Options ตัวเดียวกัน

## 2. ติดตั้งโปรแกรมครั้งแรก

แอดมินจะส่งไฟล์มาให้ 1 ใน 2 รูปแบบ — เลือกตามที่ได้รับ:

### แบบที่ 1: Installer (แนะนำลูกค้าใหม่ที่ไม่ถนัดคอม)

**Windows — `NP-Create-Setup-1.7.2.exe`**

1. ดับเบิ้ลคลิกที่ไฟล์ `.exe`
2. ถ้าขึ้น "Windows protected your PC" → กด **More info** → **Run anyway**
   - (เป็นเรื่องปกติของไฟล์ที่ยังไม่ผ่าน Microsoft notarization
     ระบบของเราไม่มีไวรัส)
3. หน้าจอติดตั้ง → กด **ถัดไป** ทุกหน้า → **ติดตั้ง** → **เสร็จสิ้น**
4. ระบบสร้าง shortcut "NP Create" ไว้บน Desktop ให้แล้ว
5. ดับเบิ้ลคลิก shortcut → ใช้งานได้เลย ไม่ต้องลง Python หรืออะไรเพิ่ม

**macOS — `NP-Create-1.7.2.dmg`**

1. ดับเบิ้ลคลิกที่ไฟล์ `.dmg`
2. หน้าต่างเปิดมา → ลาก icon **NP Create** ไปที่ **Applications**
3. เปิด Launchpad → คลิกที่ NP Create
4. ครั้งแรกถ้าขึ้นเตือน "ไม่สามารถเปิดได้":
   - System Settings → Privacy & Security → เลื่อนลงมา → **Open Anyway**
   - ครั้งต่อไปเปิดได้ปกติ

### แบบที่ 2: ZIP portable (สำหรับลูกค้า power-user)

1. **แตก zip ที่แอดมินส่งให้**
   - Windows: คลิกขวา → "Extract All..."
   - macOS: ดับเบิ้ลคลิก → ระบบจะแตกให้อัตโนมัติ
   - ได้โฟลเดอร์ชื่อ `NP-Create-customer-...`

2. **ย้ายโฟลเดอร์ไปไว้ที่ Desktop** (แนะนำ — เปิดง่าย)

3. **ดับเบิ้ลคลิกไฟล์ launcher**
   - Windows: `run.bat`
   - macOS: `run.command`
   - Linux: `run.sh`

4. **รออ่านข้อความ "กำลังติดตั้งส่วนประกอบครั้งแรก"**
   - ใช้เวลาประมาณ 30 วินาที (ขึ้นกับความเร็ว Internet)
   - หลังเสร็จ หน้าต่างโปรแกรม NP Create จะเปิดขึ้นมา

> **Windows ZIP แบบนี้ต้องลง Python 3.13 ก่อน** — ดูใน `INSTALL_TH.txt`
> ที่อยู่ในโฟลเดอร์เดียวกับ `run.bat`
>
> **macOS แสดงเตือน "ไฟล์มาจากนักพัฒนาที่ไม่รู้จัก":**
> ไปที่ System Settings → Privacy & Security → เลื่อนลงมาด้านล่าง
> จะเห็นชื่อ `run.command` พร้อมปุ่ม "Open Anyway" → กด Open

## 3. เปิดโปรแกรมและกรอก License Key

ครั้งแรก จะเห็นหน้า **Activation** ที่มีโลโก้ NP Create อยู่กลางจอ

1. กล่อง "License Key" — **วาง** คีย์ที่แอดมินส่งให้
   - คีย์มีรูปแบบประมาณ `NPC-A1B2-C3D4-E5F6-...` (5 กลุ่ม คั่นด้วย `-`)
   - **วางได้ด้วย Cmd+V (Mac) หรือ Ctrl+V (Windows)**
   - คีย์เล่ก็เล็ก ตัวใหญ่ ระบบรับได้หมด ไม่ต้องพิมพ์เป๊ะ

2. กดปุ่ม **"เปิดใช้งาน"**

3. ถ้าคีย์ถูกต้อง → เข้าหน้า Dashboard ได้เลย
   - ระบบจะจำคีย์ไว้ ครั้งต่อ ๆ ไปไม่ต้องกรอกอีก
   - 1 คีย์ใช้ได้ 3 มือถือ (ตามดีฟอลต์ — แอดมินอาจปรับได้ตอนซื้อ)

### ถ้าคีย์ไม่ผ่าน

| ข้อความ | สาเหตุ | วิธีแก้ |
|---|---|---|
| "ลายเซ็นไม่ถูกต้อง" | กดเอาคีย์ผิด หรือคีย์เก่าจากระบบเก่า | ขอคีย์ใหม่จากแอดมิน |
| "หมดอายุแล้ว" | License หมดอายุ | ติดต่อแอดมินเพื่อต่ออายุ |
| "ใช้เกินจำนวนเครื่อง" | คีย์นี้ผูกครบเครื่องแล้ว | ติดต่อแอดมินเพื่อ reset หรืออัพเกรดจำนวนเครื่อง |

## 4. เพิ่มมือถือเครื่องแรก

หน้า Dashboard ตอนยังไม่มีเครื่อง จะมีปุ่ม **"➕ เพิ่มเครื่องใหม่"**

1. **เสียบสาย USB** มือถือเข้ากับคอม
2. มือถือจะเด้งถาม **"อนุญาตการดีบัก USB?"** → กด **อนุญาต** (ติ๊ก "Always allow" ด้วย)
3. กด **"➕ เพิ่มเครื่องใหม่"**
4. หน้าต่าง Wizard จะนำคุณไปทีละขั้น
   - **Step 1:** เลือกมือถือจากลิสต์ (เช่น `Redmi 14C - 1234`)
   - **Step 2:** กด "🔧 Patch TikTok"
   - **Step 3:** รอ 1-2 นาที (ระบบจะดาวน์โหลด TikTok APK + แก้ไข + ติดตั้ง)
   - **Step 4:** เสร็จแล้ว ระบบจะเปิด TikTok บนมือถือให้อัตโนมัติ
5. **Login เข้า TikTok ตามปกติ** บนมือถือ
6. กลับมาที่คอม → กดปุ่ม **"✓ ใช้งานได้"** เพื่อบันทึกเครื่องนี้

> **ขั้น Patch อาจล้มเหลวได้ ถ้า:**
> - Internet ขาดระหว่างดาวน์โหลด → กด "Patch TikTok" ใหม่
> - มือถือกด "ปฏิเสธ" ตอนติดตั้ง → กลับไปกดอนุญาต install จาก USB อีกที
> - หน่วยความจำเต็ม → ลบรูป/แอปที่ไม่ใช้ ให้เหลือ ≥ 2 GB

## 5. เริ่มไลฟ์

หลังเพิ่มมือถือเครื่องแรกเสร็จ บน Dashboard จะเห็นการ์ด **"📹 คลิปวิดีโอ"** และ **"🎵 ไฟล์เสียง"**

### โยนคลิป

1. กดปุ่ม **"📂 เลือกคลิป"** ในการ์ดวิดีโอ
2. เลือกไฟล์วิดีโอ (`.mp4`, `.mov`, `.mkv` รองรับหมด)
   - ขนาดแนะนำ: **1080p × 1920 (แนวตั้ง)** — คมที่สุด เครื่องไหนก็รับได้
   - หรือ 720p × 1280 (แนวตั้ง) ถ้ามือถือเก่า/RAM น้อย
   - ระบบจะ encode เป็น MP4 1080×1920 อัตโนมัติ (ปรับเป็น 720p ได้ใน "ตั้งค่า → คุณภาพวิดีโอ")
   - ความยาว: ตามต้องการ ระบบจะ loop ให้

   > **ขนาดไฟล์ที่แนะนำ:** ≤ 500 MB หรือ ≤ 5 นาที
   >
   > | ขนาดคลิป | ผลที่ได้ |
   > |---------|---------|
   > | < 500 MB / < 5 นาที | ✅ Encode + push เร็ว <1 นาที, loop เนียน |
   > | 500 MB - 1 GB | ⚠️ ใช้ 1-3 นาที — ใช้งานได้ แต่ลูกค้าต้องรอ |
   > | 1 - 2 GB | ⚠️ ระบบเตือนก่อน — ใช้เวลา 5-15 นาที ต่อรอบ |
   > | > 2 GB | ❌ ไม่แนะนำ — encoder เก่ามักหมดแรม |
   >
   > **ทำไมแนะนำคลิปสั้น:** TikTok Live ดู loop เห็นชัดถ้าคลิปยาวเกิน
   > 5 นาที ลูกค้าที่กดเข้าไลฟ์มาดูจะรู้สึกเหมือนเป็นการ "เล่นซ้ำ"
   > แทนที่จะเป็นไลฟ์จริง — คลิปยาว 30 วินาที - 2 นาที loop หลายรอบ
   > จะดูเหมือนไลฟ์ตอบสนองมากกว่า
3. กดปุ่ม **"▶ Start"** ในแถบล่าง — คลิปจะเริ่มสตรีมเข้าโทรศัพท์ทันที
4. ที่มือถือ เปิด **TikTok → กดปุ่ม "+" → LIVE → เลือก "Outdoor camera"**
   - ภาพในไลฟ์คือคลิปที่คุณโยนเข้าไปนั่นเอง

### เปิดเสียงทับ

1. กดปุ่ม **"🎤 เลือกไฟล์เสียง"** ในการ์ดเสียง
2. เลือกไฟล์เสียง (`.mp3`, `.m4a`, `.wav`)
3. ระบบจะ **push ไฟล์ไปที่มือถืออัตโนมัติ** (`/sdcard/Music/`)
4. บนมือถือ เปิดแอป Music Player ใด ๆ (เช่น YouTube Music, Poweramp)
5. เล่นไฟล์ที่ชื่อ `npcreate-audio-...` ทับขณะไลฟ์
   - ตั้ง loop ไว้ก็ได้ ถ้าต้องการให้เล่นวน

## 8. เชื่อมมือถือผ่าน WiFi

หลัง Patch ครั้งแรก (ผ่าน USB) แล้ว **ครั้งต่อไปไม่ต้องเสียบสาย** ก็ได้

### ทำครั้งเดียว ตอนเสียบ USB อยู่:

1. เสียบ USB
2. บน Dashboard กดปุ่ม **"📶 เปลี่ยนเป็น WiFi"** ที่ข้างชื่อเครื่อง
3. ระบบจะ:
   - เปิด `adb tcpip 5555` บนมือถือ
   - จดจำ IP ของมือถือไว้
   - แสดง "🔌 USB → 📶 WiFi"
4. **ถอดสาย USB ได้เลย** เครื่องยังเชื่อมต่ออยู่

### ครั้งต่อไป (ไม่ต้องเสียบสาย):

- เปิดโปรแกรม
- ระบบจะลอง **auto-reconnect** มือถือทั้งหมดที่เคยใช้ WiFi
- ใช้เวลา ~3-5 วินาที จะเห็น 📶 ข้างชื่อเครื่อง

> **ข้อแม้ WiFi:**
> - คอมและมือถือต้องอยู่ WiFi เดียวกัน (Router เดียวกัน)
> - ถ้า Router restart หรือมือถือเปลี่ยน WiFi → ต้อง re-enable ผ่าน USB อีกครั้ง

## 8.5. ระบบแจ้งข่าวสาร

เมื่อแอดมินมีข่าวสำคัญ — เช่น **"TikTok อัปเดตเวอร์ชันใหม่ ต้อง Patch ซ้ำ"**
หรือ **"แจ้งปิดปรับปรุงระบบเย็นนี้"** — จะมี **กล่องข้อความสีแดง/ส้ม/ฟ้า**
โผล่ขึ้นที่ส่วนบนสุดของ Dashboard

| สี | ความสำคัญ |
|----|----------|
| 🔴 แดง | สำคัญมาก — ควรอ่านและทำตามทันที |
| 🟠 ส้ม | คำเตือน — แจ้งล่วงหน้าก่อนเกิดผลกระทบ |
| 🔵 น้ำเงิน | ข่าวทั่วไป |

- กดปุ่ม **"ปิดข้อความนี้"** ถ้าอ่านแล้ว ข้อความถัดไปจะโผล่ขึ้นแทน
- บางข้อความอาจมีปุ่มพาไปหน้าเว็บ — กดได้ตามต้องการ
- ถ้าคอมไม่ต่อ Internet ก็ใช้งานได้ปกติ แค่ไม่ได้รับข่าว

> ข้อความทุกชิ้นเซ็นด้วยลายเซ็นดิจิทัล (Ed25519) จากแอดมินเท่านั้น
> — แม้แต่ผู้ดูแล WiFi ของคุณก็ปลอมข่าวมาแสดงบนระบบไม่ได้

## 8.6. ควบคุมไลฟ์จากคอม (Live Control)

ไม่ต้องเดินไปกดที่มือถือทีละเครื่องอีกต่อไป — ที่ Dashboard มีปุ่มควบคุมไลฟ์
**สำหรับแต่ละเครื่อง** อยู่ในการ์ด **"🎬 ควบคุมไลฟ์"**

ในการ์ดนี้มี **2 ปุ่ม** ใช้แตกต่างกันตามสถานการณ์

| ปุ่ม | ใช้เมื่อ |
|------|---------|
| **🔴 เริ่มไลฟ์** (ปุ่มใหญ่สีแดง) | ใช้แบบ **อัตโนมัติ** — ระบบกด TikTok ให้เอง |
| **▶ จับเวลาเอง** (ปุ่มเล็กใต้ปุ่มแดง) | ใช้แบบ **manual** — คุณกด TikTok เอง ระบบนับเวลาให้อย่างเดียว |

### โหมด A: เริ่มไลฟ์อัตโนมัติ (ปกติ)

1. เลือกเครื่องที่ Sidebar ซ้าย
2. กดปุ่มแดง **"🔴 เริ่มไลฟ์"**
3. ระบบจะถามยืนยัน → กด "Yes"
4. รอ 15-30 วินาที — ระบบจะกดให้เองอัตโนมัติ:
   - เปิด TikTok
   - **กด `[+]` (Create button)**
   - ไปแท็บ LIVE
   - กด Go Live → Screen Share → Start Now
5. **ไลฟ์เริ่มแล้ว** — ตัวจับเวลาเริ่มเดิน `🔴 ไลฟ์อยู่ — 00:01:23`

ระหว่างทำงาน ป้ายสถานะข้างปุ่มจะอัปเดต **real-time** ทุกขั้นตอน — เห็นว่ากดอะไรไปแล้ว
ถ้าค้างก็รู้ว่าค้างที่ไหน

### โหมด B: จับเวลาเอง (manual)

ใช้ในกรณี:
- TikTok เปลี่ยน layout ทำให้ระบบ auto หาปุ่มไม่เจอ (เกิดขึ้นได้ทุก 2-4 สัปดาห์)
- คุณชอบกด TikTok เอง แต่อยากบันทึกระยะเวลาไลฟ์ไว้บนคอม
- เครื่องที่ Go Live เป็นไอคอนล้วนๆ ไม่มี text label

วิธีใช้:

1. เปิด TikTok บนเครื่อง → กด Live → Screen Share → Start Now **เอง**
2. กลับมาที่คอม → กดปุ่มเล็กๆ **"▶ จับเวลาเอง (ฉันกดไลฟ์เองบนเครื่อง)"**
3. ยืนยัน → ตัวจับเวลาเริ่มทันที — `🔴 ไลฟ์อยู่ — 00:00 (เริ่มเอง)`
4. จบไลฟ์ในมือถือเอง → กลับมากด **"⏸ หยุดจับเวลาเอง"** บนคอม

> **ข้อดีของโหมด manual:** ไม่ขึ้นกับเวอร์ชัน TikTok เลย ทำงานได้ 100% เสมอ

### โหมด C: Recovery (เมื่อ auto fail แต่ไลฟ์เริ่มแล้ว)

ถ้ากด "🔴 เริ่มไลฟ์" แล้ว auto ค้างที่ step ใด step หนึ่ง (เช่น "✗ Go-Live button not found")
ระบบจะ **popup ถาม** ว่า

> "ไลฟ์เริ่มแล้วใช่ไหม?
> ถ้าตอนนี้คุณกดไลฟ์เองบนเครื่อง [...] แล้ว — กด Yes
> เพื่อเริ่มจับเวลาบน PC"

- **กด Yes** ถ้าไลฟ์เริ่มแล้วจริง (auto กดไม่ครบแต่คุณกดต่อเอง) → timer เริ่มทันที
- **กด No** ถ้ายังไม่ได้เริ่ม → ลองใหม่ / ใช้โหมด manual แทน

### จบไลฟ์ (Auto stop)

1. กดปุ่ม **"⏹ จบไลฟ์"**
2. ระบบจะกดปุ่มย้อน + กด "ยืนยัน" บนมือถือให้
3. ตัวจับเวลาบันทึกระยะเวลา → ปรากฏที่ **"ไลฟ์รวม: X:XX ชม."** ด้านขวา

> **ถ้าคุณใช้โหมด manual เริ่ม** ก็ใช้ปุ่ม **"⏸ หยุดจับเวลาเอง"** เพื่อจบ —
> ไม่ต้องผ่านระบบ auto

### ใช้งานหลายเครื่องพร้อมกัน

- ที่ Sidebar เครื่องที่ไลฟ์อยู่จะมี **`🔴 LIVE 25:43`** ติดต่อท้ายชื่อ
- กดสลับไปดูทีละเครื่อง — ตัวจับเวลาบนการ์ดหลักเปลี่ยนตามที่เลือก
- การปิดโปรแกรมระหว่างไลฟ์ **ไม่ทำให้ตัวจับเวลาหายไป** — เปิดมาใหม่ก็ยังนับต่อ

### ตารางความหมายของข้อความที่อาจเจอ

| ข้อความ | สาเหตุ | วิธีแก้ |
|---------|--------|--------|
| "→ tapped Create [+]" / "→ tapped Live tab" / "→ tapped Go Live" | **ปกติ** กำลังเดินตามขั้นตอน | รอต่อ |
| "✗ Go-Live button not found" | TikTok เปลี่ยน layout / บัญชียังไลฟ์ไม่ได้ | popup recovery จะถามต่อ — กด Yes ถ้ากดเองแล้ว / ใช้โหมด manual |
| "❌ บัญชีนี้ยังไลฟ์ไม่ได้" | follower < 1,000 / ยังไม่ยืนยันอายุ | ทำให้ครบเงื่อนไขก่อน หรือใช้บัญชี alt |
| "❌ Screen Share ไม่พร้อม" | บางภูมิภาคปิดฟีเจอร์ | ใช้โหมด manual + ใช้ camera live (ไม่ใช่ screen share) |
| "⚠️ จบไลฟ์ด้วยการบังคับปิดแอป" | หาปุ่ม End ไม่เจอ ระบบใช้ force-stop | ครั้งหน้ากด End ในมือถือเองก่อน แล้วกด "หยุดจับเวลา" บนคอม |

### กรณีที่จบไลฟ์ไม่ได้แบบสะอาด

ถ้าหา "ปุ่ม End live" ในมือถือไม่เจอ ระบบจะใช้ **Force-stop** เป็น fallback —
ผู้ชมจะหลุดทันทีแบบไม่บอกล่วงหน้า มีข้อความเตือน *"⚠️ Force-stop"* ขึ้นเพื่อให้
รู้ว่าครั้งหน้าควรกด End ในมือถือเองดีกว่า

## 8.7. Mirror หน้าจอเครื่อง (คุมผ่าน PC)

> **🆕 ฟีเจอร์ใหม่ v1.7.3** — วางมือถือเฉยๆ ทำงานทุกอย่างจาก PC ได้เลย

ใต้การ์ด **"🎬 ควบคุมไลฟ์"** จะมีปุ่ม **"🪞 Mirror หน้าจอเครื่อง (คุมผ่าน PC)"**
กดแล้วจะ:

1. เปิดหน้าต่างใหม่ที่แสดงหน้าจอมือถือแบบ realtime (delay ~50 มิลลิวินาที)
2. **คลิก / เลื่อน / พิมพ์** ในหน้าต่างได้เหมือนแตะที่หน้าจอจริง
3. **หน้าจอมือถือดับ** เพื่อประหยัดแบต — แต่ Mirror ยังทำงานปกติ
4. มือถือจะ **ไม่หลับ** ตลอดเวลาที่ Mirror เปิดอยู่

### ใช้ทำอะไรได้บ้าง

- ตั้งค่าไลฟ์ (พิมพ์ชื่อ live, เลือก category, เลือกหน้าปก) ผ่านคีย์บอร์ด PC
- ตอบคอมเมนต์ระหว่างไลฟ์ — พิมพ์เร็วกว่ามือถือ 5 เท่า
- เปิด/ปิดเสียงไมค์ผ่านปุ่มบนหน้าต่าง mirror
- ดูตัวอย่างก่อนเริ่ม Live โดยไม่ต้องหยิบมือถือ
- เลื่อนดู For You Page ระหว่างพักหายใจ

### วิธีใช้ครั้งแรก

**กรณีปกติ** (ใน ZIP มี scrcpy bundled แล้ว — ลูกค้าใหม่ทุกคน):

1. กดปุ่ม **"🪞 Mirror หน้าจอเครื่อง"** ที่การ์ด "🎬 ควบคุมไลฟ์"
2. หน้าต่างมือถือเด้งมาภายใน 1-2 วินาที — **ไม่ต้องดาวน์โหลดอะไรเลย**
3. ลากไปวางที่ฝั่งซ้าย/ขวาของจอ PC

**กรณีฉุกเฉิน** (ใช้ ZIP เก่า / ไฟล์ scrcpy หาย):

1. กดปุ่ม **"🪞 Mirror"** → ระบบจะถามว่าจะติดตั้งให้อัตโนมัติไหม?
2. กด **"ใช่"** → ดาวน์โหลด ~9 MB จาก github.com (มี progress bar) → ติดตั้งใน ~30 วินาที
3. **ไม่ต้องเปิด Terminal / PowerShell / ติด admin อะไรทั้งสิ้น** — แอปทำให้
4. ครั้งต่อๆ ไปจะเปิดทันที (ไม่ต้องดาวน์โหลดซ้ำ)

> **ที่อยู่ไฟล์** (ถ้าอยากเช็ค/ลบ):
> - **bundled** (มาพร้อม ZIP): `<โฟลเดอร์ NP Create>/.tools/<os>/scrcpy/scrcpy`
> - **auto-installed** (กรณีฉุกเฉิน): `~/.npcreate/tools/scrcpy-3.x.x/` (macOS/Linux)
>   หรือ `C:\Users\<ชื่อ>\.npcreate\tools\scrcpy-3.x.x\` (Windows)

| **Ctrl+Shift+H** | ปุ่ม Home |
| **Ctrl+Shift+F** | Fullscreen |
| **คลิก × ที่หน้าต่าง** | ปิด mirror (ปุ่มในแอปจะกลับเป็น "🪞 Mirror" ทันที) |

### ระบบรู้สถานะตลอดเวลา

ปุ่มใน NP Create จะเปลี่ยนตามสถานะ:

| สถานะปุ่ม | ความหมาย |
|-----------|----------|
| **🪞 Mirror หน้าจอเครื่อง (คุมผ่าน PC)** | พร้อมเปิด — กดได้เลย |
| **🪞 Mirror (ติดตั้งครั้งแรก ~9 MB)** | ครั้งแรก — กดเพื่อให้ระบบดาวน์โหลด+ติดตั้ง scrcpy ให้อัตโนมัติ |
| **⏹ ปิด Mirror** (พื้นเขียว) | กำลังเปิด Mirror อยู่ — กดเพื่อปิด |
| **🪞 Mirror หน้าจอเครื่อง** (จาง) | เครื่อง offline — เชื่อมต่อก่อน |

ใต้ปุ่มจะมีข้อความบอก **"✓ Mirror เปิดอยู่ (12 นาที) — หน้าจอมือถือถูกปิดเพื่อประหยัดแบต"**

### ข้อสังเกต

- ใช้ผ่าน **USB จะลื่นที่สุด** (delay ~30ms) — ผ่าน WiFi ADB จะ ~80-150ms
- เปิด Mirror หลายเครื่องพร้อมกันได้ — แค่กดสลับเครื่องที่ Sidebar แล้วกด Mirror
- ปิด NP Create แล้วทุก Mirror window จะปิดเองอัตโนมัติ
- ถ้าเสียบสายแล้วเครื่อง **ขออนุญาต USB Debugging อีกครั้ง** ที่หน้าจอมือถือ → กด Allow ก่อนถึงจะ Mirror ได้

> **เคล็ดลับ:** ใช้ Mirror **ก่อน** กดปุ่ม "🔴 เริ่มไลฟ์" เพื่อตั้งค่าหน้าปก / ชื่อไลฟ์ให้พร้อม
> แล้วค่อยกดปุ่มแดงให้ระบบเริ่มไลฟ์อัตโนมัติ — ทำได้ทุกอย่างโดยไม่ต้องหยิบมือถือเลย

## 8.9. ส่ง Log ให้แอดมิน (Diagnostic Export)

เมื่อเจอบั๊กที่อธิบายยาก ตรงไป **ตั้งค่า → ความช่วยเหลือ** แล้วกด:

- **"📋 สร้าง / ส่ง Log ให้แอดมิน"** — ระบบจะรวมไฟล์ log + ข้อมูลเครื่อง + การตั้งค่า
  ลงใน ZIP เดียว ส่งให้แอดมินทาง Line OA ได้เลย
- **"📂 เปิดโฟลเดอร์ Log"** — ดู / copy ไฟล์เองตามต้องการ

### ข้อมูลที่อยู่ใน ZIP

| ไฟล์ | คืออะไร |
|------|---------|
| `system_info.json` | OS, Python version, app version, ADB version |
| `logs/npcreate.log*` | บันทึกการทำงานล่าสุด ~20 MB |
| `config.redacted.json` | การตั้งค่า (License Key ถูกปกปิด) |
| `activation.redacted.json` | สถานะ License (Key ถูกปกปิด) |
| `customer_devices.json` | รายการเครื่องที่ pair ไว้ |

### สิ่งที่ระบบ **ไม่มี** ใน ZIP (ปลอดภัย):

- ❌ License Key ของคุณ — ถูกแทนด้วย `<redacted:24-chars>`
- ❌ TikTok Shop Token / รหัสผ่านอื่นๆ — ถูกปกปิดทั้งหมด
- ❌ Private Signing Key — ไม่มีในเครื่อง user อยู่แล้ว

## 8.9. สำรอง / กู้คืน (Backup & Restore)

> ** ใช้เวลาตอนไหน?** ตอนเปลี่ยนคอม / format Windows ใหม่ /
> ติดตั้งซ้ำ — ไม่ต้อง pair มือถือใหม่ทุกเครื่อง

### สำรอง (Backup)

1. ตั้งค่า → **"💾 สำรอง / กู้คืนการตั้งค่า"** → กด **"💾 สำรอง (Backup)"**
2. ระบบจะถามที่เซฟ ZIP — แนะนำเก็บใน USB Drive / Google Drive / OneDrive
3. ขนาดไฟล์ปกติ ~10-50 KB

### กู้คืน (Restore)

1. ลงโปรแกรม NP Create บนคอมใหม่ตามขั้นตอนติดตั้งปกติ
2. เปิดโปรแกรม → กรอก License Key (ใช้ Key เดิม)
3. ตั้งค่า → **กด "📥 กู้คืน (Restore)"** → เลือก ZIP ที่สำรองไว้
4. ยืนยัน → **ปิด-เปิดโปรแกรมใหม่** = ใช้งานได้เลย

### สิ่งที่ Backup เก็บ
- ✅ License (activation บนเครื่องเดิม)
- ✅ รายการมือถือที่ pair ไว้ (พร้อม WiFi IP, ชื่อเล่น, การหมุนภาพ)
- ✅ การตั้งค่า encode (1080p / 720p / mirror)
- ✅ ประวัติ License (ฝั่งแอดมิน)

### สิ่งที่ Backup **ไม่** เก็บ
- ❌ ตัวโปรแกรม (ลงใหม่จากตัวติดตั้ง)
- ❌ FFmpeg / JDK / ADB (ลงใหม่อัตโนมัติ)
- ❌ Private Signing Key

## 8.10. อัปเดตอัตโนมัติ (Auto-update)

เวลามีเวอร์ชันใหม่ จะมี **กล่องสีแดง "🚀 อัปเดตใหม่ vX.Y.Z พร้อมใช้งาน"** โผล่ที่ Dashboard
พร้อมรายการสิ่งที่เปลี่ยนแปลง

### วิธีใช้

1. กดปุ่ม **"⬇️ อัปเดตเลย"**
2. รอแถบดาวน์โหลด (ปกติ ~10-30 วินาที — แพทช์มีขนาดเล็ก ~500 KB เท่านั้น)
3. ระบบจะติดตั้งและ **รีสตาร์ทตัวเองอัตโนมัติ**
4. เปิดมาใหม่ = ใช้เวอร์ชันใหม่ทันที

### ข้อดี

- **ไม่ต้องโหลด ZIP ใหม่ทั้งก้อน 250 MB ทุกครั้ง** — ดาวน์โหลดเฉพาะส่วนที่เปลี่ยน
- ของเดิมเก็บไว้เป็น `src.bak/` เผื่อ rollback ถ้าเวอร์ชันใหม่มีปัญหา
- ตรวจสอบลายเซ็นดิจิทัล Ed25519 ก่อนติดตั้ง — ป้องกันแพทช์ปลอม

### กรณีพิเศษ — "ดาวน์โหลดตัวติดตั้ง"

บางอัปเดต (เช่น เปลี่ยน FFmpeg / JDK ที่ฝังในโปรแกรม) ปุ่มจะกลายเป็น **"🌐 ดาวน์โหลดตัวติดตั้ง"**
- กดแล้วเว็บจะเปิดให้ดาวน์โหลด `.exe` / `.dmg` ใหม่
- ติดตั้งทับของเดิมได้เลย (Setting/License/Devices ทั้งหมดเก็บไว้ที่ `%APPDATA%\NPCreate` ไม่หายไปไหน)

### ถ้าอัปเดตล้มเหลว

- กดปุ่ม **"⬇️ ลองอัปเดตอีกครั้ง"** ที่จะโผล่ขึ้นอัตโนมัติ
- ถ้ายังไม่ได้ → ลองปิด-เปิดโปรแกรมใหม่ แล้วลองอีกครั้ง
- กรณีฉุกเฉิน: ลบโฟลเดอร์ `src/` ทิ้ง แล้ว rename `src.bak/` → `src/` (ย้อนกลับเวอร์ชันเดิม)

### ระบบเช็คอัปเดตเมื่อไหร่?

- **ทุก 6 ชั่วโมง** ขณะเปิดโปรแกรม
- ถ้าไม่อยากอัปเดตตอนนี้ กดปุ่ม **"ภายหลัง"** — กล่องจะหายไป กลับมาใหม่ครั้งหน้าที่เปิดโปรแกรม

## 9. Troubleshooting

### "Patched ล้มเหลว" / Patch ค้างนาน

1. เช็ค Internet — ระบบต้องโหลด TikTok APK ~120 MB
2. เช็คพื้นที่มือถือ ≥ 2 GB
3. ลองถอด-เสียบ USB ใหม่
4. ลองรีสตาร์ทคอม + มือถือ แล้ว Patch ใหม่
5. ส่ง log ให้แอดมิน: ในโปรแกรม Cog icon → "Export log"

### "ไม่เจอเครื่อง" บน Dashboard

| สาเหตุ | วิธีเช็ค |
|---|---|
| USB Debugging ปิดอยู่ | เช็คในมือถือ: ตั้งค่า → ตัวเลือกนักพัฒนา |
| ไม่ได้กด "Allow" บนมือถือ | ดูที่หน้าจอมือถือ มี popup ขึ้น |
| สาย USB เสีย / ไม่ใช่สาย data | ลองสายอื่น (สายชาร์จบางเส้นโอนข้อมูลไม่ได้) |
| Driver USB Windows | ลงไดรเวอร์มือถือ (Xiaomi USB Driver, Samsung USB Driver) |

### TikTok เปิดแล้วเด้งออก / Login ไม่ได้

- ลอง **uninstall TikTok ออก แล้ว Patch ใหม่ทั้งระบบ**
- ตรวจ Android version ≥ 9.0
- ถ้าเป็นเครื่องที่ root อยู่ ต้องปิด Magisk Hide ก่อน

### โปรแกรมเปิดไม่ขึ้น (กด run.bat แล้วไม่มีอะไรเกิดขึ้น)

- เปิด Command Prompt → ลาก `run.bat` มาใส่ → กด Enter
- จะเห็น error message → ส่งภาพให้แอดมิน
- หรือดู log ที่ `%TEMP%\NP-Create-install.log` (Windows) / `$TMPDIR/NP-Create-install.log` (Mac)

### ภาพในไลฟ์เป็นจอดำ / ภาพหยุดนิ่ง

1. กด **Stop** แล้ว **Start** ใหม่
2. ตรวจสอบว่าเลือก camera "**Outdoor**" ในเมนูไลฟ์ของ TikTok (ไม่ใช่กล้องหน้า/หลัง)
3. ลอง encoder อื่น: ในตั้งค่า → "ตัวเข้ารหัสวิดีโอ" → ลองเปลี่ยนเป็น `libx264`

### Encode ค้าง / ใช้เวลานานมาก

มักเกิดจากคลิปต้นฉบับใหญ่เกินไป (≥ 1 GB) หรือยาวเกิน ~10 นาที

1. **ตรวจขนาดไฟล์** — แนะนำ ≤ 500 MB หรือ ≤ 5 นาที
2. ตัดให้สั้นลง (ใช้ CapCut / Premiere / Davinci ตัด export ใหม่)
3. ปรับเป็น 720p ใน "ตั้งค่า → คุณภาพวิดีโอ" ถ้าคอมเก่า
4. ปิดโปรแกรมหนักอื่น (Chrome แท็บเยอะ ๆ, OBS ฯลฯ) ระหว่าง encode
5. ถ้าคลิป > 1 GB ระบบจะถาม **"ไฟล์ใหญ่ ดำเนินการต่อ?"** ก่อนเริ่ม
   เพื่อกันไม่ให้รอเงียบ ๆ โดยไม่รู้ตัว

> **ทำไม TikTok Live ชอบคลิปสั้น:** loop ของคลิปสั้นจะดูเหมือนไลฟ์
> มากกว่าคลิปยาว — ผู้ชมที่เลื่อนเข้ามาดูจะไม่เห็นรอยต่อ

### ภาพในไลฟ์สะท้อนเป็นกระจก (ตัวอักษร/โลโก้กลับด้าน)

- โดยปกติระบบจะ pre-flip ให้อัตโนมัติ (default เปิด)
- ถ้ายังเห็นกลับด้าน → ตั้งค่า → **คุณภาพวิดีโอ** → toggle "แก้ภาพสะท้อนอัตโนมัติ"
- ถ้าตั้งค่าเปิดแล้วยังกลับด้าน ลองปิดดู (บางเครื่องที่ใช้ rear camera ไม่ต้อง flip)
- จากนั้นกด **Encode + Push** ใหม่หนึ่งครั้ง

### เสียงไม่ออกในไลฟ์

- ระบบไม่ได้ inject เสียงเข้า TikTok โดยตรง (TikTok ใช้ native audio path)
- **ต้องเล่นไฟล์เสียงผ่าน Music Player บนมือถือ** ตามที่อธิบายไว้ใน [ส่วน 5](#5-เริ่มไลฟ์)
- เช็ค volume ของแอป Music Player ไม่ใช่ ringtone volume

### 9.7 TikTok ถูกอัปเดตเองในเบื้องหลัง — vcam หาย

> **อาการ:** เคย Patch สำเร็จแล้วใช้ไลฟ์ได้ปกติมาหลายวัน — วันนึง
> เปิดไลฟ์ ภาพกลับมาเป็นกล้องหน้า/หลังเหมือนไม่เคย Patch หรือ
> โปรแกรม NP Create เด้งเตือน **"⚠️ TikTok ถูกอัปเดต"**

**สาเหตุ:** TikTok เด้ง popup "อัปเดตเวอร์ชันใหม่" ในแอป
แล้วผู้ใช้กด OK โดยไม่รู้ว่ามันจะลบ vcam ของเราทิ้ง — เวอร์ชันใหม่
ที่ดาวน์โหลดมาจาก Play Store / TikTok server **ไม่มี LSPatch**
ของเราอยู่แล้ว vcam จึงหายทันที

**วิธีแก้ (ทำได้เองในโปรแกรม v1.7.5+):**

1. เปิด NP Create — ระบบจะตรวจเจอว่าเวอร์ชัน TikTok ในมือถือ
   ไม่ตรงกับเวอร์ชันที่ Patch ไว้ และจะเด้งหน้าต่าง
   **"⚠️ TikTok ถูกอัปเดต"** มาให้
2. กด **"ใช่ — Re-Patch ตอนนี้"** ในหน้าต่างนั้น
3. **เสียบสาย USB ไว้** (Re-Patch ต้องใช้ USB ครั้งเดียว)
4. รอ ~1 นาที ระบบจะ pull TikTok เวอร์ชันใหม่ + patch + ติดตั้ง
5. เสร็จแล้วกลับมาใช้งานได้ปกติ

**วิธีกันไม่ให้เกิดอีก:**

- **อย่ากด "อัปเดต"** ในหน้าจอ TikTok ที่เด้งขึ้นเฉย ๆ — กด
  "ภายหลัง" / "ปิด" ทุกครั้ง
- **ปิด Auto-Update** ของ TikTok ใน Play Store:
  - เปิด Play Store → กด profile (มุมขวาบน) → จัดการแอป
    และอุปกรณ์ → จัดการ → กด TikTok → กด "⋮" → **ปิด** "เปิด
    การอัปเดตอัตโนมัติ"
- ในมือถือ Xiaomi/Huawei/OPPO ที่มี App Store ของตัวเอง ก็ปิด
  auto-update ของ TikTok ใน store นั้นด้วย

> **เกร็ด:** ระบบจะเด้งเตือน Re-Patch แค่ครั้งเดียวต่อรอบเปิด
> โปรแกรม จะไม่กวนซ้ำ แม้คุณยังไม่ได้ Re-Patch ก็ตาม

### 9.8 TikTok บังคับ "ต้องอัปเดต" ก่อนเริ่มไลฟ์

> **อาการ:** กดปุ่ม **"เริ่มไลฟ์"** ใน TikTok แล้วเด้ง popup
> **"ต้องอัปเดตเวอร์ชันถึงจะ Live ได้"** — ปิดไม่ได้ ไม่กดก็ไม่
> ขึ้นไลฟ์

**สาเหตุ:** TikTok ตรวจเวอร์ชันที่ฝั่ง **server** ตอนเปิด Live
ไม่ใช่แค่ใน app — ทุก ๆ ~1-2 เดือน TikTok จะปรับ "เวอร์ชันต่ำสุด
ที่ไลฟ์ได้" ขึ้น ถ้าเวอร์ชันที่ Patch ไว้ต่ำกว่ามาตรฐานนั้น
**บังคับ update เท่านั้น** ไม่มีทางเลือกอื่น

> **คนละเคสกับ 9.7** — เคส 9.7 คือ TikTok เด้ง popup ทั่วไปในแอป
> (กดปิดได้). เคส 9.8 คือ TikTok ปฏิเสธไลฟ์ของคุณจาก server เลย
> (ปิดไม่ได้ ต้องอัปเดต)

**วิธีแก้ — ทำตามขั้นตอนใน NP Create v1.7.5+:**

1. **บนมือถือ** — กด "อัปเดต" ตามที่ TikTok บอก (vcam จะหาย
   ชั่วคราว ไม่ต้องตกใจ)
2. รอจน Play Store ติดตั้ง TikTok ใหม่เสร็จ — ตรวจดู: เปิด
   TikTok แล้วไม่มี popup update อีก
3. **เสียบสาย USB** เครื่องนี้กับคอม
4. ในโปรแกรม NP Create — กดปุ่ม
   **"🆙 TikTok บังคับ update ก่อนไลฟ์? — กดที่นี่"**
   (อยู่ใต้ปุ่ม Patch ที่หน้า Dashboard)
5. กด "ใช่" ในหน้าต่างยืนยัน — ระบบจะ:
   - ดึง TikTok เวอร์ชันใหม่จากมือถือ (~30 วิ)
   - patch ใส่ vcam
   - ติดตั้งทับให้อัตโนมัติ
6. เสร็จแล้วเปิด TikTok → กด "เริ่มไลฟ์" ได้เหมือนเดิม

> **เวลาที่ใช้รวม:** ~1-2 นาที (ขึ้นอยู่กับ Internet มือถือ +
> ความเร็วของ Play Store)

> **คนละบัญชีไหม?** ไม่ — Re-Patch ไม่ได้ลบ TikTok ออกแล้วลงใหม่
> มันแค่ "ติดตั้งทับ" — บัญชีที่ login อยู่ คงเดิม

## 10. FAQ

**Q: ใช้กับ TikTok บัญชีอื่น ๆ ในเครื่องเดียวกันได้ไหม?**
A: ได้ ไม่จำกัดบัญชี แค่ Logout-Login สลับได้ปกติ

**Q: ใช้พร้อมกัน 3 เครื่อง 3 บัญชี ได้ไหม?**
A: ได้ครับ License 1 คีย์รองรับ 3 เครื่อง พร้อมกันได้ทั้ง 3

**Q: คลิปต้องเตรียมยังไง?**
A:
- แนวตั้ง 720×1280 หรือ 1080×1920 = ไหลลื่นที่สุด
- แนวนอน ก็ได้ แต่จะมีขอบดำในไลฟ์
- ความยาว: ระบบ loop อัตโนมัติ ใช้คลิป 30 วินาที วน 1 ชั่วโมงได้สบาย

**Q: ใช้ได้ทั้ง Windows + Mac ในคีย์เดียวกันไหม?**
A: ได้ — คีย์ผูกกับ "**จำนวนมือถือ**" ไม่ใช่จำนวนคอม

**Q: คีย์หาย ต้องทำยังไง?**
A: ทักแอดมิน ระบบเก็บประวัติคีย์ทุกใบที่ออกไป สามารถส่งคีย์เดิมให้คุณใหม่ได้

**Q: ต่ออายุยังไง?**
A: ก่อนหมดอายุ 7 วัน แอดมินจะส่งข้อความเตือน — โอนเงินค่าต่ออายุ ได้คีย์ใหม่ทันที (คีย์เก่ายังใช้ได้จนวันหมดอายุ)

**Q: เพิ่มจำนวนเครื่องระหว่างเดือนได้ไหม?**
A: ได้ — แจ้งแอดมิน ออกคีย์ใหม่ที่จำนวนเครื่องมากขึ้นให้ คีย์เก่าจะถูก revoke

**Q: TikTok update เวอร์ชันใหม่ จะใช้ได้ไหม?**
A: ระบบ Patch แบบ dynamic — รองรับ TikTok เวอร์ชันใหม่ได้เกือบทั้งหมด
ถ้ามีเคสใช้ไม่ได้ แจ้งแอดมินจะออก patch ให้ทันที

**สำคัญ — มี 2 เคสคนละแบบ ต้องแยกให้ออก:**

- *เคสปกติ:* TikTok เด้ง popup ชวนอัปเดต (กดปิดได้) →
  **อย่ากด** อัปเดต. ถ้าเผลอกด ใช้ปุ่ม Re-Patch ที่ NP Create
  เด้งให้ ([9.7](#97-tiktok-ถูกอัปเดตเองในเบื้องหลัง--vcam-หาย))

- *เคสบังคับ:* กด "เริ่มไลฟ์" แล้ว TikTok บอก "ต้องอัปเดตเวอร์ชัน
  ถึงจะ Live ได้" (ปิดไม่ได้) → **กดอัปเดต** ตามที่ TikTok บอก →
  จากนั้นกลับมา NP Create กดปุ่ม **"🆙 TikTok บังคับ update
  ก่อนไลฟ์?"** ([9.8](#98-tiktok-บังคับ-ต้องอัปเดต-ก่อนเริ่มไลฟ์))

**Q: ทำไมเครื่องเดียวกันต้อง Re-Patch บ่อย?**
A: เพราะ TikTok ค่อย ๆ ปรับ "เวอร์ชันต่ำสุดที่ไลฟ์ได้" ขึ้นทุก
~1-2 เดือน ทำให้เวอร์ชันที่เคย patch ใช้ไม่ได้กับ Live API
แล้ว — Re-Patch ทับด้วยเวอร์ชันใหม่ก็ใช้งานต่อได้ปกติ
ไม่ต้องลบ login เก่า

**Q: ปลอดภัยไหม? โดนบัญชีแบนหรือเปล่า?**
A: ระบบนี้ไม่ได้ใช้ root และไม่ได้แก้ไขบัญชี TikTok โอกาสโดน detect น้อยมาก แต่ไม่มีระบบใดในโลกที่รับประกัน 100% — แนะนำใช้กับบัญชีที่ไม่ใช่บัญชีหลัก
